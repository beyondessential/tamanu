Workhorse product development flow

The canonical happy path from a freshly created card to shipped, verified work. Each step is one phase; the agent and the user collaborate throughout, and changes auto-commit on every turn.

Phase

Agent

Cross-cutting

  • Auto-commitEvery agent turn and user edit commits to the card's branch — no manual saves.
  • SkillsNamed prompts (interview, mock this up, draft spec, review, implement) shape the agent's behaviour for a turn.
  • HandoffWhen the agent hits a capability limit, it offers a handoff prompt for an external tool rather than looping.
  • Spec explorerMerged specs from the main branch form a navigable hierarchy that informs every future card.
